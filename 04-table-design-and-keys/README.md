# 04 — Table Design and Primary Keys

This section focuses on improving table structure by introducing primary keys and unique identifiers.

## What’s covered
- adding a primary key to an existing table
- using `AUTO_INCREMENT` to generate unique identifiers
- understanding why a primary key must be `NOT NULL` and `UNIQUE`
- inspecting table structure before and after changes

## Files
- `alter_primary_key.sql` — SQL examples showing how to add an auto-increment primary key to an existing table using `ALTER TABLE`

## Notes
This section focuses on table design and structure. Data modification and advanced `ALTER TABLE` operations are covered in later sections.
